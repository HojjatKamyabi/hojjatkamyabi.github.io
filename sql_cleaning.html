Motor Vehicle Repair and Towing Data Cleaning using SQL

Github Repository


Overview

For this project, I worked on cleaning and analyzing a dataset related to Motor Vehicle Repair and Towing services. The dataset contained inconsistencies such as duplicate records, incorrect formatting, missing values, and non-standardized data. Using SQL, I performed various data cleaning operations to ensure accuracy and reliability for further analysis.


Steps Taken


1. Data Staging

Before making any modifications to the original dataset, I created a staging table as a backup to preserve the raw data. This allowed for safe transformations without affecting the source data.

2. Removing Duplicates

To identify and remove duplicate records, I used a Common Table Expression (CTE) to assign row numbers based on duplicate groups. Then, I created a secondary staging table where only unique records were retained.

3. Standardizing Data

To improve consistency, I applied formatting rules such as trimming whitespace, correcting state abbreviations, and converting date fields from text to date format.

4. Handling Missing Values

To address missing values, I first replaced blank fields with NULL and then used self-joins to fill missing corporation names based on matching trade names.

5. Exploratory Data Analysis (EDA)

After cleaning the dataset, I conducted exploratory analysis to extract insights, such as:

- The number of repair businesses per city

- Monthly trend of issued registrations

- Ranking corporations by number of issued registrations per year


Conclusion

Through this project, I demonstrated my ability to clean and prepare real-world data using SQL. The final dataset was free of duplicates, standardized, and structured for meaningful analysis.
